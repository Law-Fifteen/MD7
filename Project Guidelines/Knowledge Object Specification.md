# Philosophy

Everything inside the platform is a Knowledge Object.

Not a page.

Not a chapter.

Every object should be reusable.

# Future Compatibility

Knowledge Objects should be database-friendly.

Avoid embedding relationships directly.

Relationships should reference IDs.

The platform should eventually support thousands of Knowledge Objects.

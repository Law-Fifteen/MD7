# MD7 Sales Academy
## UX Interaction Standards

---

# Purpose

This document defines how the application should feel.

It does not describe appearance.

It describes behavior.

Every interaction should feel intentional, polished, and consistent.

The experience should resemble modern premium software rather than a traditional learning management system.

---

# Overall UX Philosophy

The application should feel:

• Calm

• Professional

• Executive

• Responsive

• Fluid

• Minimal

Never overwhelming.

Never cluttered.

Never distracting.

---

# Reading Experience

Reading should feel effortless.

Maximum content width.

Comfortable typography.

Generous spacing.

Sticky chapter navigation.

Estimated reading time.

Reading progress indicator.

Chapter completion indicator.

Automatically remember reading position.

---

# Progress Indicators

Every major section should display progress.

Overall Progress

↓

Course Progress

↓

Chapter Progress

↓

Section Progress

↓

Exercise Progress

Progress should animate smoothly.

---

# Navigation Behavior

Sidebar remains persistent.

Current chapter remains highlighted.

Recently viewed content is remembered.

Related content is always visible.

Breadcrumb navigation on every page.

Back navigation should never lose scroll position.

---

# Cards

Cards slightly elevate on hover.

Cards expand smoothly.

Cards should never feel static.

Clickable cards always indicate interaction.

---

# Accordions

Default collapsed.

Animate open.

Remember previous state.

Nested accordions should be avoided.

---

# Tabs

Use tabs only when comparing related information.

Avoid more than five tabs.

Remember selected tab.

---

# Hover Behavior

Hover should communicate possibility.

Never surprise.

Never trigger large layout shifts.

Micro animations only.

---

# Buttons

Buttons animate subtly.

Primary actions visually dominate.

Secondary actions remain available but quieter.

Loading states must exist.

Disabled states must explain why.

---

# Knowledge Checks

Immediate feedback.

Explain reasoning.

Reference relevant chapters.

Allow retry.

Track completion.

---

# Reflection

Reflection sections should feel personal.

Use expandable text areas.

Allow saving locally.

Never require completion.

---

# Search

Instant.

Fuzzy matching.

Grouped results.

Keyboard shortcuts.

Highlighted search terms.

Recent searches.

Popular searches.

---

# Empty States

Every empty state should educate.

Never display:

"No results."

Instead display:

Suggested searches.

Related chapters.

Alternative resources.

---

# Success States

Completion should feel rewarding.

Use subtle celebration.

Avoid confetti.

Display:

Progress updated

Competency updated

Suggested next lesson

---

# Motion

Animations:

150–250ms

Smooth easing

No bounce

No unnecessary delay

Motion supports comprehension.

---

# Mobile

Bottom navigation.

Large touch targets.

Swipe between chapters.

Sticky progress.

Collapsible sidebar.

---

# Accessibility

Keyboard navigation.

Visible focus states.

Screen reader labels.

High contrast.

Reduced motion support.

Accessible color contrast.

Every interaction should be usable without a mouse.
